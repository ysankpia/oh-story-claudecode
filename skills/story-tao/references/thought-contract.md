# 思想命题产物协议

`设定/思想命题.md` 是长篇写作与审稿消费思想层的唯一权威产物。候选讨论、原典全文和命题卡都不是项目事实。

## 权威与优先级

只有同时满足以下条件才可消费：

1. 文件存在且可读；
2. YAML 头包含 `status: confirmed`；
3. `primary_operator` 指向现有命题卡；
4. 必填段落完整。

缺失、未确认或损坏时输出修复提示并忽略思想层，不阻断写作。优先级固定为：用户明确要求与锁定细纲/读者契约 > 思想命题 > 题材取舍与文风。思想命题不得新增细纲外事件。

## 日更摘要

日更只抽取与本章相关的四项，不传完整文件：

- 当前人物立场；
- 本章对应的命题检验；
- 本章选择、代价与后果；
- 读者契约兼容和表达禁区。

摘要只校准既有细纲的意义与人物选择。若需要新增事件才能完成命题检验，标记 `thought_alignment_conflict`，遵循细纲继续写，不自动补纲。

## 审稿规则

思想完整性是独立建议，不计入原总分、严重级别或发布结论。检查：

1. 命题是否由人物行动和后果体现；
2. 反命题是否被公平呈现；
3. 角色是否沦为观点传声筒；
4. 原典是否误引或断章取义；
5. 思想层是否破坏题材承诺、主角代理权或既定文风。
